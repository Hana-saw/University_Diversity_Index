## はじめる前に

### GitHubアカウントの作成
まだアカウントをお持ちでない方は [github.com/signup](https://github.com/signup) から無料で作成できます。

### データの信頼性について
**必ず公式な情報源に基づいてデータを入力してください。**
- ✅ 各大学の公式Webサイト
- ✅ 大学が発行する年次報告書・IR情報
- ✅ 文部科学省の公開統計データ
- ✅ 政府機関による調査結果
- ❌ まとめサイト・個人ブログ
- ❌ 情報源が不明な数値

## データを修正する（既存の大学）

### 方法A：GitHubのWeb UIから直接編集（最も簡単）

1. 修正したい大学の詳細ページを開く
2. **「GitHubで編集・提案する」** ボタンをクリック
3. GitHubのファイル編集画面が開くので、内容を修正する
4. ページ下部の **「Propose changes」** をクリック
5. 変更内容の説明を入力して **「Create pull request」** をクリック

> 💡 GitHubにログインしていない場合は、ログイン後に自動的にフォークが作成されます。

### 方法B：ローカルで編集

```bash
# 1. リポジトリをフォーク（GitHubのUIから）

# 2. フォークをクローン
git clone https://github.com/あなたのユーザー名/University_Diversity_Index.git
cd University_Diversity_Index

# 3. 依存パッケージをインストール
npm install

# 4. 開発サーバーを起動
npm run dev

# 5. ブランチを作成
git checkout -b fix/update-tokyo-university

# 6. ファイルを編集
# src/content/universities/university-of-tokyo.md を修正

# 7. コミット＆プッシュ
git add .
git commit -m "fix: 東京大学の女性教授比率を最新データに更新"
git push origin fix/update-tokyo-university

# 8. GitHubからPull Requestを作成
```

## 新しい大学を追加する

### ファイル名のルール

- 英語の大学名をハイフン区切りの小文字で命名
- 例：`kyushu-university.md`、`nagoya-university.md`
- スペースや日本語は使わない
- `src/content/universities/` フォルダに配置

### テンプレート

以下をコピーして新しい `.md` ファイルを作成してください：

```markdown
---
name: 大学名（日本語）
name_en: University Name (English)
region: 地域（関東 / 関西 / 東北 / 中部 / 中国・四国 / 九州 / 北海道・東北 など）
prefecture: 都道府県名（例：東京都、大阪府、愛知県）
gender_ratio_female_prof: 数値（例：18.5）  # 0〜100の数値、%表記なし
lgbtq_policy: 実施済み  # 「実施済み」「検討中」「未着手」の3択
lgbtq_policy_detail: |
  LGBTQ+施策の具体的な内容を記述。
  複数行にわたる場合はインデントを揃えて記述する。
barrier_free_score: 3  # 1〜5の整数
barrier_free_detail: |
  バリアフリー状況の具体的な説明を記述。
international_ratio: 10.0  # 省略可能。外国人教員比率（0〜100）
official_url: https://www.example.ac.jp  # 大学公式サイトのURL
github_id: university-name  # ファイル名と同じ（拡張子なし）
last_updated: "2024-03-01"  # データを確認した日付（YYYY-MM-DD形式）
data_source: 出典元の名称（例：○○大学 男女共同参画推進室 2023年度報告書）
---

## 概要

大学の概要とダイバーシティに関する全体的な方針・姿勢を記述します。

## 女性教授比率について

女性教授比率の現状、背景、目標値などを記述します。

## LGBTQ+施策

具体的な施策内容、相談窓口の有無、学生団体の活動状況などを記述します。

## バリアフリー状況

各キャンパスの整備状況、支援体制、今後の計画などを記述します。
```

## Markdownファイルの書き方

### Frontmatterの各フィールドの詳細

| フィールド | 型 | 必須 | 説明 |
|------------|-----|------|------|
| `name` | 文字列 | ✅ | 大学の正式名称（日本語） |
| `name_en` | 文字列 | | 英語名称 |
| `region` | 文字列 | ✅ | 地域区分（下記参照） |
| `prefecture` | 文字列 | ✅ | 所在都道府県（「東京都」「大阪府」など） |
| `gender_ratio_female_prof` | 数値 | ✅ | 女性教授比率（0〜100の小数） |
| `lgbtq_policy` | 選択肢 | ✅ | `実施済み` / `検討中` / `未着手` |
| `lgbtq_policy_detail` | 文字列 | | 施策の詳細説明 |
| `barrier_free_score` | 整数 | ✅ | バリアフリースコア（1〜5） |
| `barrier_free_detail` | 文字列 | | バリアフリーの詳細説明 |
| `international_ratio` | 数値 | | 外国人教員比率（省略可） |
| `official_url` | URL | ✅ | 公式サイトURL（https://〜） |
| `github_id` | 文字列 | ✅ | ファイル名（拡張子なし） |
| `last_updated` | 日付 | ✅ | `"YYYY-MM-DD"` 形式 |
| `data_source` | 文字列 | | データの出典 |

### 地域区分の一覧

| region に入力する値 | 対象都道府県 |
|---------------------|------------|
| `北海道・東北` | 北海道、青森、岩手、宮城、秋田、山形、福島 |
| `関東` | 東京、神奈川、埼玉、千葉、茨城、栃木、群馬 |
| `中部` | 新潟、富山、石川、福井、山梨、長野、岐阜、静岡、愛知 |
| `関西` | 三重、滋賀、京都、大阪、兵庫、奈良、和歌山 |
| `中国・四国` | 鳥取、島根、岡山、広島、山口、徳島、香川、愛媛、高知 |
| `九州・沖縄` | 福岡、佐賀、長崎、熊本、大分、宮崎、鹿児島、沖縄 |

### バリアフリースコアの基準

| スコア | 目安 |
|--------|------|
| 5 | 全キャンパスで完全整備済み。UDデザインを全面採用。支援体制も充実。 |
| 4 | ほぼ整備済み。一部建物に軽微な課題はあるが、代替手段がある。 |
| 3 | 主要建物は整備済みだが、古い建物はアクセスに制約あり。 |
| 2 | 整備が進行中だが、移動に支障をきたす場面が多い。 |
| 1 | 整備がほとんどされておらず、大幅な改修が必要な状態。 |

## Pull Requestの送り方

### PRタイトルの書き方

```
種別: 大学名 - 変更内容の概要
```

**種別の例：**
- `fix:` データの誤り修正
- `update:` データの最新化
- `add:` 新規大学の追加
- `docs:` ドキュメントの更新

**タイトル例：**
```
fix: 東京大学 - 女性教授比率を2023年度データに更新
add: 名古屋大学 - 新規追加
update: 京都大学 - LGBTQ+施策を「実施済み」に更新
```

### PR本文に含めること

```markdown
## 変更内容
<!-- 何を変更したか簡潔に説明してください -->

## 情報源
<!-- 変更に使用したデータの出典URLを記載してください -->
- URL: https://...
- 資料名: 〇〇大学 IR情報 2023年度版

## 変更前 → 変更後
<!-- 数値変更の場合は必ず記載してください -->
- 女性教授比率: 16.7% → 18.2%

## 備考
<!-- その他、レビュアーに伝えたいことがあれば -->
```

## データ品質のガイドライン

### ✅ 良いデータの条件

- 公式の一次情報源（大学公式サイト、IR報告書、文部科学省統計）に基づいている
- `last_updated` が情報を確認した日付になっている
- `data_source` に出典が明記されている
- 数値は半角数字で入力されている

### ❌ 避けてほしいこと

- 情報源を確認せずに数値を入力・変更すること
- 個人の推測や意見をデータとして記載すること
- 出典不明の数値を採用すること
- 他サイトのコンテンツをそのままコピーすること

### 数値が見つからない場合

- 比率が不明な場合は、文部科学省の「学校基本調査」や大学のIR情報を参照してください
- それでも見つからない場合は、PRコメントに「要調査」と記載の上でPRを送っていただければ、コミュニティで一緒に探します

## よくある質問

**Q. プログラミングができなくても貢献できますか？**
A. はい。データの修正や追加はMarkdown（プレーンテキスト）の編集なので、プログラミング知識は不要です。

**Q. どの大学を追加してもいいですか？**
A. 基本的に日本の大学・大学院であれば歓迎します。ただし、データの根拠となる公式情報源が存在することが条件です。

**Q. 情報が古くなっていますが、最新データが見つかりません。**
A. Issueを作成して「〇〇大学のデータが古い」と報告するだけでもOKです。コミュニティが一緒に探します。

**Q. PRがマージされるまでどのくらいかかりますか？**
A. 通常1〜7日程度を目安にレビューします。データの根拠が明確なPRは優先的にマージします。

**Q. 誤ってデータを上書きしてしまいました。**
A. GitHubの履歴からいつでも復元できるので、心配しないでください。
